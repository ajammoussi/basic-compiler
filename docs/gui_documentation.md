# Basic Compiler GUI Documentation

## Overview

The Basic Compiler GUI is an interactive PyQt5 application that provides visual representations of all compilation phases. It integrates seamlessly with the backend compiler logic and offers real-time feedback on source code analysis.

## Features

### 1. Lexer Tab

**Purpose**: Visualize lexical analysis (tokenization)

**Features**:
- Source code editor with syntax highlighting
- Token stream table showing:
  - Line number
  - Column position
  - Token type
  - Token value
- Color-coded tokens:
  - Keywords (if, then, else, int, string): Blue, bold
  - Identifiers: Green
  - Numbers: Red
  - String literals: Magenta
  - Operators: Orange
  - Delimiters: Gray
- Error reporting for invalid characters

**How to Use**:
1. Enter source code in the editor
2. Click "Tokenize" button
3. View the generated token stream in the table

### 2. Parser Tab

**Purpose**: Visualize syntax analysis and AST construction

**Features**:
- Source code editor
- Interactive parse tree (AST) visualization
- Tree nodes show:
  - Node type (e.g., Assignment, BinaryExpression, Identifier)
  - Node attributes (name, value, operator, etc.)
- Expandable/collapsible tree nodes
- Error reporting for syntax errors

**How to Use**:
1. Enter source code in the editor
2. Click "Parse" button
3. Explore the generated AST by expanding/collapsing nodes

### 3. Semantic Analyzer Tab

**Purpose**: Visualize semantic analysis, type checking, and scope resolution

**Features**:
- Source code editor
- Symbol table showing:
  - Scope level
  - Variable name
  - Variable type
  - Declaration line/column
- Semantic errors table showing:
  - Line number
  - Column position
  - Error message
- Real-time type checking feedback

**How to Use**:
1. Enter source code in the editor
2. Click "Analyze" button
3. View the symbol table and any semantic errors

### 4. Pipeline Tab

**Purpose**: Visualize the complete compilation pipeline

**Features**:
- Source code editor
- Pipeline visualization showing:
  - Lexical Analysis phase status
  - Syntax Analysis phase status
  - Semantic Analysis phase status
  - Success/failure indicators
  - Detailed information for each phase
- Color-coded status:
  - Green: Success
  - Red: Failed
- Expandable error details

**How to Use**:
1. Enter source code in the editor
2. Click "Compile" button
3. View the complete compilation pipeline status

### 5. Example Templates

**Available Examples**:
- Basic assignment: Simple variable assignment
- Arithmetic: Complex arithmetic expressions
- If-else (braces): Conditional logic with blocks
- Inline if-else: Conditional logic without blocks
- While loop: Iterative loop visualization
- String operations: Basic string variable handling
- String concatenation: Demonstrates "string + string"
- Type mismatch error: Demonstrates type checking (int vs string)
- Undeclared variable: Demonstrates scope checking
- Duplicate declaration: Demonstrates error handling for re-declared variables

**How to Use**:
1. Select an example from the dropdown
2. The source code loads into the current tab
3. Run the appropriate analysis (Tokenize, Parse, Analyze, or Compile)

## Visual Design

The GUI uses a dark theme inspired by modern IDEs:
- Background: Dark gray (#1e1e1e)
- Text: Light gray (#d4d4d4)
- Accent color: Blue (#0078d4)
- Success: Green (#00FF00)
- Error: Red (#FF0000)

## Integration with Backend

The GUI directly uses the backend compiler components:

- **Lexer**: `src.lexer.tokenize()`
- **Parser**: `src.parser.parse()`
- **Semantic Analyzer**: `src.semantic_analyzer.SemanticAnalyzer`
- **Symbol Table**: `src.symbol_table.SymbolTable`
- **Error Handler**: `src.error_handler.ErrorHandler`

All error messages, tokens, and AST nodes are displayed exactly as generated by the backend.

## Keyboard Shortcuts

- **Ctrl+Enter**: Run the current tab's action (Tokenize/Parse/Analyze/Compile)
- **Ctrl+L**: Load example from dropdown
- **Ctrl+C**: Copy selected text
- **Ctrl+V**: Paste text

## Troubleshooting

### GUI won't start

**Problem**: PyQt5 not installed

**Solution**:
```bash
pip install PyQt5
```

### Syntax highlighting not working

**Problem**: Tokens not being highlighted

**Solution**: Click "Tokenize" first to generate tokens, then syntax highlighting will be applied.

### Parse tree not showing

**Problem**: Empty tree after parsing

**Solution**: Check for syntax errors in the error messages. The tree only shows if parsing succeeds.
